# Rust Image Viewer

Rust + Tauri + Vue 3 による軽量デスクトップ画像ビューアです。フォルダ・ZIP・RAR アーカイブから画像を読み込み、高速に表示します。MangaMeeyaの代替として作成し、使用しています。

## 特長

- **高画質表示**: ブラウザネイティブ形式(JPEG/PNG/WebP/AVIF/GIF)は元画像をそのまま WebView2 の GPU スケーリングへ渡し、サーバーサイド再エンコードによる世代ロスを完全排除
- **高DPI対応**: `devicePixelRatio` を考慮し、Windows のディスプレイスケーリング環境でも鮮明に表示
- **高速応答**: 非同期先読みキャッシュ + LRU エビクションでページ送りが軽快、かつ長時間閲覧でもメモリ使用量が増大しない
- **堅牢なセキュリティ**: アーカイブ爆弾 / デコード爆弾 / シンボリックリンク攻撃に対する制限を組み込み
- **最小権限設定**: Tauri の capability は `core` / `dialog` / フルスクリーン操作のみに制限

## 動作環境

- Windows 10 / 11 (x64)
- Rust 1.94+
- Node.js 22+
- npm 11+

## 対応画像形式

| 形式 | 拡張子 | レンダリング方式 |
|------|--------|------------------|
| JPEG | `.jpg` `.jpeg` | ブラウザネイティブ(元画像パススルー) |
| PNG | `.png` | ブラウザネイティブ(元画像パススルー) |
| WebP | `.webp` | ブラウザネイティブ(元画像パススルー) |
| AVIF | `.avif` | ブラウザネイティブ(元画像パススルー) |
| GIF | `.gif` | ブラウザネイティブ(元画像パススルー) |
| BMP | `.bmp` | Rust 側でデコード → リサイズ → 再エンコード |

ブラウザネイティブ形式は Rust 側でデコード・再エンコードせず、base64 data URL で元のバイト列をそのまま転送し、WebView2(Chromium) の GPU スケーリングに任せます。これにより JPEG の世代ロスが完全に排除され、読み込みも高速化します。

## 対応ソース

- **フォルダ**: 指定したディレクトリ以下を再帰的に走査(シンボリックリンクは追跡しない)
- **ZIP アーカイブ**: `.zip` ファイル内の画像を直接読み込み
- **RAR アーカイブ**: `.rar` ファイル内の画像を直接読み込み

いずれもファイル名を自然順(numerical sort)で並び替えます(`1.jpg, 2.jpg, 10.jpg, 100.jpg` の順)。ソートキーは Schwartzian transform (`slice::sort_by_cached_key`) で一度だけ計算し、大規模フォルダでも高速に処理します。

## 操作方法

### ファイルの読み込み

| 操作 | 説明 |
|------|------|
| ドラッグ＆ドロップ | アプリ内にフォルダ/ZIP/RAR をドロップ。**画像表示中にもドロップ可能**で、新しいソースに完全に切り替わります |
| クリックでフォルダ選択 | 起動画面のドロップゾーンをクリックしてフォルダ選択ダイアログを開く |
| アーカイブ選択 | 起動画面の「アーカイブ選択」リンクから ZIP/RAR ファイルを選択 |
| コマンドライン | `rust-image-viewer.exe "path/to/file.zip"` |
| exe に D&D | エクスプローラから `rust-image-viewer.exe` にファイルをドロップして起動 |

新しいソースを読み込んだ際、現在のページインデックスは先頭(1枚目)にリセットされます。

### ページ操作

| 操作 | デフォルトキー | 備考 |
|------|----------------|------|
| 次のページ | `←` | 最後のページで折り返し |
| 前のページ | `→` | 最初のページで折り返し |
| 最初のページ | `Home` | |
| 最後のページ | `End` | |
| 左クリック | 次のページ | |
| 右クリック | 前のページ | コンテキストメニューは無効 |
| マウスホイール | ページ送り | 上＝前、下＝次 |

### 表示切替

| 操作 | デフォルトキー | 備考 |
|------|----------------|------|
| 全画面 / ウィンドウ切替 | `Enter` | 全画面時はスライダー・ステータスバーを非表示、DPR を考慮して鮮明に再レンダリング |
| 単ページ / 見開き切替 | `Space` | 見開きは 2 ページ同時表示 |
| 設定パネル | `Esc` | トグル動作(開く/閉じる) |

### ページスライダー

画面下部に全ページ数のスライダーを配置。ドラッグで任意のページにジャンプできます。スライダー左側に `⏮`(最初へ)、`⏭`(最後へ)のボタンがあります。

## 設定

7 種類のアクションに対してキーを割り当てられます。項目をクリックしてキーを押すことで変更できます。修飾キー(Ctrl / Alt / Shift)との組み合わせも対応しています。

### 表示

|------|------|

### 動作

| 項目 | デフォルト値 | 説明 |
|------|-------------|------|
| 先読み枚数 | 10 | 現在のページの前後に事前読み込みする枚数 |

### 起動時の状態復元

- 表示モード(単ページ / 見開き)

## 主な設計方針

### 描画パイプライン

- **ブラウザネイティブ形式のパススルー**: JPEG / PNG / WebP / AVIF / GIF は Rust 側でデコード・再エンコードせず、元のバイト列を base64 data URL で転送。GPU スケーリングにより最高画質と高速読み込みを両立
- **高DPI対応レンダリング**: `devicePixelRatio`(上限 2.0) を乗算した解像度で画像を要求し、CSS ピクセル不足によるボヤけを防止
- **フルスクリーン時の UI オフセット調整**: フルスクリーンではスライダー・ステータスバー分の高さ(56px)を減算せず、全画面解像度に合わせて再フェッチ
- **BMP のみ Rust サイドで処理**: ブラウザ非ネイティブ形式の BMP は必要に応じて CatmullRom フィルタでリサイズし、PNG(リサイズ無し)または JPEG(リサイズ時)にエンコードして転送

### キャッシュと先読み

- **世代管理**: 非同期画像取得に世代カウンターを用い、古いリクエスト結果は破棄して表示崩れを防止
- **先読みキャッシュ**: 現在のページの前後を非同期で先読みし、スムーズなページ送りを実現(先読み枚数は設定可能)
- **LRU エビクション**: キャッシュは最大30エントリ。`currentIndex` から最も遠いものから破棄し、長時間閲覧でもメモリ使用量を抑制(画像サイズによるが、概ね 100〜300MB 程度で頭打ち)

### 操作性

- **入力スロットリング**: キーボード・マウスホイールのスロットリング間隔を設定可能、操作詰まりを防止
- **全画面切替の即時追従**: キャッシュ世代を進めつつ古い画像は表示し続け、新しい解像度の画像取得後にシームレスに差し替え
- **見開き表示の最適化**: 左画像を右寄せ・右画像を左寄せに配置し、綴じ目での隙間を解消
- **D&D 常時受付**: リスナを `App.vue` のライフサイクルに紐付け、起動画面・表示中のどちらでも新しいソースをドロップ可能

### 永続化と後方互換性

- **設定の後方互換性**: `#[serde(default)]` により、新しいバージョンでフィールドが追加・削除されても古い設定ファイルを壊さない(未知のフィールドは無視、欠落フィールドはデフォルト補完)
- **起動時の状態復元**: ウィンドウサイズと表示モードを保存・復元

## セキュリティ

### アーカイブ・デコード制限

| 制限 | 値 | 目的 |
|------|-----|------|
| ZIP/RAR 最大エントリ数 | 10,000 | 列挙時の DoS 防止 |
| ZIP/RAR 1エントリ最大サイズ | 100 MB | 展開時のメモリ枯渇防止 |
| 画像 raw データ上限 | 150 MB | デコード前のメモリ保護 |
| 画像デコード最大次元 | 40,000 × 40,000 px | デコード爆弾防止 |
| 画像デコード最大割当 | 512 MB | `image` クレートの `Limits` で強制 |

### ファイルシステム

- **シンボリックリンク非追跡**: `walkdir::WalkDir::follow_links(false)` で循環参照や意図しない領域への再帰を防止

### Tauri 権限

`capabilities/default.json` は以下の最小権限のみ許可:

- `core:default` (Tauri コア)
- `dialog:default` / `dialog:allow-open` (ファイル選択ダイアログ)
- `core:window:allow-set-fullscreen` / `core:window:allow-is-fullscreen`

外部 URL オープン、ファイル書込、シェル実行等の権限は付与していません。

### CSP

Content Security Policy により XSS 攻撃を防止:

```
default-src 'self';
img-src data: 'self';
style-src 'self' 'unsafe-inline';
```

`img-src data:` は base64 data URL による画像表示に必要。`'unsafe-inline'` は Vue のスタイル付与に必要。
